# 自動ドキュメンテーション生成プロンプト（Docs用）

## 目的
対象ソースコード内の**すべての関数**に対し、明確で一貫性のあるドキュメンテーションコメントをつける
AIのトークン制約を考慮しつつ、**最大5関数ずつ自動的にTodoに分割して、全関数を連続処理**する。

## 指示
1. 対象ソースコードを解析し、**関数ごとにドキュメンテーションコメントを生成**する。  
2. コメントは関数定義の直前に挿入する。  
3. 関数が5個を超える場合は、**自動的に5関数ずつ順次処理し、全関数完了まで繰り返す。**
   - ユーザーからの「次の5関数を処理」といった追加指示は絶対に不要。
　 - 全ての関数にドキュメンテーションコメントをつけるまではタスクを完了しない  
   - AIが内部的にバッチ処理を継続し、すべての関数にコメントを付与する。  
4. コメント内容には以下を含める：  
   - 関数の目的（何をするか）  
   - 引数（型・説明）  
   - 戻り値（型・説明）  
   - 副作用（該当する場合）  
5. コメント形式は対象言語の標準形式に従う：  
   - **C#** → XMLドキュメントコメント (`/// <summary>...</summary>`)  
   - **JavaScript / TypeScript** → JSDoc (`/** ... */`)  
   - **Python** → Docstring (`""" ... """`)  
   - **Go** → 通常コメント (`// ...`)  

## 制約
- コメントは1関数あたり3〜6行に収める  
- コードロジックは変更しない  
- 関数名・引数名はそのまま使用  
- コメントは日本語で簡潔・明確に  
- AIは処理を中断せず、関数が尽きるまで自動で全件処理する  

## 期待する出力形式
```markdown
### 処理対象コード全体（5関数単位で内部処理しつつ最終出力は全関数含む）

```csharp
/// <summary>
/// 2つの数値を加算します。
/// </summary>
/// <param name="a">最初の数値</param>
/// <param name="b">2番目の数値</param>
/// <returns>加算結果</returns>
public int Add(int a, int b)
{
    return a + b;
}

/// <summary>
/// 2つの数値を乗算します。
/// </summary>
/// <param name="a">最初の数値</param>
/// <param name="b">2番目の数値</param>
/// <returns>乗算結果</returns>
public int Multiply(int a, int b)
{
    return a * b;
}
```

### 処理フロー（AI内部）
1. ソースコード内の全関数を検出  
2. 5関数ずつドキュメント生成を実行  
3. 内部的に次の5関数を自動的に続行  
4. 最終的にすべての関数にコメントが追加されたコードを出力  

## 入力・出力例

### 入力
```python
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b
```

### 出力
```python
def add(a, b):
    """
    2つの数値を加算します。

    Args:
        a (int | float): 最初の数値。
        b (int | float): 2番目の数値。

    Returns:
        int | float: 加算結果。
    """
    return a + b


def multiply(a, b):
    """
    2つの数値を乗算します。

    Args:
        a (int | float): 最初の数値。
        b (int | float): 2番目の数値。

    Returns:
        int | float: 乗算結果。
    """
    return a * b
```
