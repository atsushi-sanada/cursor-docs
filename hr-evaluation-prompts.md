# 人事評価プロンプト

## 概要

- プロジェクトファイルを参照して人事評価をお願いします。

## **評価仕様**

### 目的

Slack の投稿ログと自己・多面評価をもとに、**S〜G ランクの定量評価**と、**職種ベースの実務に即した定性コメント**を出力する。

---

## 評価ロジック（定量）

### 入力データ

#### 一覧

以下の３つのデータから評価をしてください。全てプロジェクトファイルに保存しています。

| データソース     | 評価項目の強化例                 |
| ---------------- | -------------------------------- |
| 自己評価記述     | 主体性 or 認識ギャップ判定       |
| 多面評価コメント | 調整力・共感力などの補正加点     |
| Slack の投稿ログ | 稼働安定性 or タスク完遂度の評価 |

#### 入力データについて

- 自己評価記述 : 自分の自己評価です。未提出も可なのでない場合もあります。
- 多面評価コメント : 他のメンバーの評価です。人によって数が違います。未提出も可なのでない場合もあります。
- Slack 投稿ログ : 各メンバーの投稿ログの json が保存されています。
  - Json のデータ構造
  ```
  [
      {
          "timestamp": "文字列",  // Unix 時間（秒）を小数点以下付きの文字列で表現
          "channel": "文字列",    // チャンネル ID または名前
          "text": "文字列"        // メッセージ本文（改行やリンクを含むことがある）
      }
  ]
  ```

### 基本構成（5 パラメータ）

| パラメータ | Slack 上の言動指標           | スコア化方法（最大 100 点）                                                                                                                                                                                                                                                         |
| ---------- | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **行動力** | 投稿数（頻度）               | 以下の実数ベースでスコアを段階的に設定：<br>・`投稿数 ≥ 1000`：100 点<br>・`500〜999`：`投稿数 ÷ 1000 × 100`<br>・`100〜499`：`投稿数 ÷ 500 × 80`（最大 80 点）<br>・`0〜99`：`投稿数 ÷ 100 × 50`（最大 50 点）<br>Slack 上の活動実態に基づき、偏りを反映しやすくしたスケーリング。 |
| **発信力** | 情報共有・納品投稿           | 「共有します」「提出しました」「〜お送りします」などの投稿件数をカウント。最大 30 件で 100 点に正規化。                                                                                                                                                                             |
| **調整力** | レビュー・修正・コメント投稿 | 「ご確認ください」「調整お願いします」など確認・調整・レビュー行動、および周囲との認識合わせ・進行調整の行動全般を含む投稿をカウントし、最大 30 件で 100 点に正規化。<br>Slack 外での調整貢献が多面評価等で確認される場合、+5〜+15 の補正加点を行う。                               |
| **共感力** | 感謝・労いの表現投稿         | 「ありがとうございます」「助かります」「お疲れ様です」などを含む投稿件数をカウントし、最大 30 件で 100 点に正規化。<br>多面評価に「安心感」「丁寧」「雰囲気が良い」などの表現がある場合、+5〜+10 の補正を加点。                                                                     |
| **主体性** | 質問・提案・確認投稿         | 「どうしましょうか？」「提案ですが」「気になる点があります」など、起点的・自発的な発言を抽出し、最大 30 件で 100 点に正規化。<br>Slack 上に発言が少なくても、自己・多面評価に「率先」「自走」などの記述があれば、+5〜+15 の補正加点を行う。                                         |
|  |

→ 各スコアを 0〜100 に正規化し、以下でランク化：

| スコア範囲 | ランク |
| ---------- | ------ |
| 90〜100    | S      |
| 75〜89     | A      |
| 60〜74     | B      |
| 45〜59     | C      |
| 30〜44     | D      |
| 15〜29     | E      |
| 0〜14      | G      |

➡ 各パラメータにランクを付与し、**総合ランクは平均スコアのランクで決定**。

## 評価コメント生成ロジック（定性）

### 出力構成

- **評価ポイント（ポジティブ）×2 行**
- **期待ポイント（ネガティブ）×1 行**

### コメント構成方針

- 実際の投稿内容（例：「★ 印コメント」「FIX 確認」など）を活用し、職種ベースの貢献を簡潔に表現
- Slack 投稿と自己・多面評価の一致度を見て補足観点を加える
- 表現トーンは「真田風」に整える（簡潔・現場視点・丁寧な口調）

## 書き出しフォーマット

- Google スプレッドシートに貼り付けら得るように以下の列構成で書き出してください。
  名前 総合評価 行動力 発言力 調整力 共感力 主体性 評価ポイント 期待ポイント
- 評価値は数字で書き出してください。ランクはあとで関数でつけます。
- そのまま貼り付けたいので書き出す値に見出しは不要です。
